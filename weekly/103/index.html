本周讨论：大脑不是思考器官，而是预测器官

（本期来源：得到头条 156 期 | “压力好大，好想吃垃圾食品”，怎么破？）

前不久，《神经元》杂志上刊登了一项研究，这是由澳大利亚新南威尔士大学医院的团队完成的。奇点网对此进行了解读转载。研究的主题是：为何人们在压力下，会更加偏爱垃圾食品？

当我们面临压力，比如加班、失恋或遭遇挫折时，我们往往会更倾向于吃炸鸡、比萨、冰激凌等食物。如果你在网上搜索“失恋爆肥”，你会发现一堆相关的结果，其中包括许多明星的例子。而且，吃垃圾食品似乎真的能让我们的心情变得更好，否则为何有人会把可乐称为“肥宅快乐水”呢？

但是，这件事仔细想想，其实挺奇怪的。要知道，人的所有本能，都是经过了千百万年的进化。只有那些对生存有利的本能，才会被保留下来。

比如，看见毒蛇就害怕，能帮我们避开危险。看见苹果就开心，能推动我们获取食物。但是，压力大的时候就想吃垃圾食品，这明显不合理。你看，压力已经很大了，再继续吃垃圾食品，不是雪上加霜吗？我们的身体，为什么要保留这么一个对自己不利的设置呢？

按照过去的设想，这就是自己没毅力。但是，这回科学家有了一个新发现。人之所以在压力大的时候爱吃垃圾食品，是因为大脑里面，有一个脑区被异常关闭。

这个脑区，叫做外侧缰核，它的作用是给我们泼冷水，让我们不要沉迷于某件事。比如，你看电视剧看了很久，就会觉得有点腻了。这就是外侧缰核在提醒你，该干点别的事了。

同理，当你吃高脂食物的时候，外侧缰核也会在适当的时候告诉你，够了，别再吃了。但问题是，外侧缰核很怕压力。如果你长期处于压力状态下，它就会关闭自己。这样一来，你就没有人给你泼冷水了。你就会一直吃下去，越吃越开心。

所以说，不要怪自己没毅力。其实是压力改变了你的大脑结构。而这个过程，你根本无法感知和控制。

听到这里，你可能会觉得有点无奈。我们的大脑真的那么不靠谱吗？它为什么要做出这样不合理的反应呢？

有一位著名的神经科学家，叫莉莎·费德曼·巴瑞特。她是美国艺术与科学院，还有加拿大皇家学会的双院士。她曾经写过一本书，叫《认识大脑》，去年年底刚出中文版。

她在书里提出了一个非常反常的观点，她说，大脑根本就不是一个思考器官，而是一个预测器官。它的主要功能，是预测可能发生的事。

比如说，当你渴了的时候喝水，你会立刻觉得解渴了。但其实水需要 20 分钟才能进入血液。那为什么会立刻解渴呢？因为大脑预测到你喝水之后会解渴，并提前给你释放信号。

再比如说，当你跑马拉松冲过终点时，你会觉得精疲力尽。但其实你身体里还有很多能量储备。那为什么会觉得精疲力尽呢？因为大脑预测到你可能还需要逃生，并提前给你保留能量。

这些预测都是基于进化赋予我们的本能反应。它们并不一定正确或合理。但它们都是为了帮助我们管理身体预算（body budget），也就是身体能量和资源的分配和平衡。

这一切都是本能反应，你根本感觉不到过程。就像一加一等于二，计算过程太快，以至于你都意识不到自己计算过。我们只能感受到它的结果，也就是我们的感觉和情绪。我们以为我们是根据感觉和情绪来思考和行动，但其实是大脑根据预测来调节我们的感觉和情绪。

那么，既然大脑的预测机制有时候会出错，我们有没有办法改变它呢？答案是肯定的。因为预测是基于过去的经验，所以我们可以通过改变经验来改变预测。

比如说，你可以尝试在心情不好的时候，不吃垃圾食品，而是去跑步或者做其他运动。这样一来，你就给大脑提供了一个新的经验，让它知道运动也可以让你开心。多做几次，大脑就会形成一个新的预测回路。下次你心情不好的时候，你就会想去运动，而不是吃垃圾食品。

再比如说，你可以尝试在加班之后，给自己一个奖励，比如玩游戏、看电影、吃美食等。这样一来，你就给大脑提供了一个新的经验，让它知道加班也可以带来乐趣。多做几次，大脑就会形成一个新的预测回路。下次你加班的时候，你就会感觉到一点兴奋，而不是压抑。

总之，人类是一个非常奇妙的物种。我们有很多本能反应，有些是对我们有利的，有些是对我们不利的。但我们也有能力通过改变经验来改变本能反应。有句话说得好，成大事者，反本能。拿什么对抗本能？答案是，激发另一个本能。

2、世界上最大的办公楼（英文）

位于印度古吉拉特邦苏拉特市的新开放的苏拉特钻石交易所，这是一个为超过 65000 名钻石专业人士提供“一站式目的地”的地方，包括切割工、抛光工和交易员。

文章

1、人工智能的风险是真实但可控的（英文）

来自于比尔·盖茨的个人博客。

详细讨论了人工智能（AI）带来的风险，并坚称这些风险是可以管理的。他提到，尽管 AI 创新带来了新的威胁，但这并不是我们首次面临这样的挑战。无论是汽车的引入，还是个人电脑和互联网的崛起，人们都成功地管理了其他的转型时刻，并最终在许多动荡之后得到了改善。

他详细讨论了 AI 带来的各种风险，包括深度伪造和 AI 产生的误导信息可能破坏选举和民主，AI 使得对人和政府的攻击更加容易，AI 可能会取代人们的工作，AI 可能继承我们的偏见并编造事实，以及学生可能因为 AI 的帮助而不再学习写作。然而，他也指出，这些问题都有历史先例，我们可以从过去的经验中学习，而且许多由AI引起的问题也可以通过 AI 来解决。

2、如何编写出色的自述文件（英文）

这篇文章探讨了如何编写一个出色的项目 README。README
文件是你的项目的前门——它通常是你给用户和贡献者的第一印象。一个有效的 README 文件需要告诉你的受众你的项目做什么，如何使用它，以及他们如何帮助。

文章中详细讲解了一个好的 README 应遵循的原则，包括从最有用的信息开始，然后逐渐深入到技术细节；尽可能避免大段文字，而是链接到其他源以获取更多详细信息；并尽可能使用视觉效果。文章还详细解析了 Appsmith 的 README，并提供了其他优秀 README 的例子。

3、提示工程学的开发人员指南（英文）

这篇文章是 GitHub 的博客文章，主要讲述了如何使用大型语言模型（LLM）进行提示工程。文章首先解释了 LLM 的工作原理，然后详细介绍了如何使用 LLM 构建应用程序，以及如何优化提示工程技术。文章还详细解释了 GitHub Copilot 代码补全的工作原理，以及如何使用 LLM 提高代码补全的效率和准确性。

4、超越内向与外向（英文）

这篇文章是关于人际交往的一种新理解。作者认为，将人们简单地划分为内向和外向的二元模型已经无法满足我们对人际交往的理解。

他提出了一个新的理念，即人们在不同的环境下会有不同的交往方式，他将这些环境划分为四种状态：独处、一对一、小组和大群体。

每个人在这四种状态下的交往方式和需求都会有所不同，这就构成了每个人的交往模型。通过理解和调整自己的交往模型，我们可以更好地理解自己，改善人际关系，提高生活质量。

5、塑造软件开发的革命性 IDE（英文）

这篇文章回顾了过去 30 年中最具影响力的 10 款集成开发环境（IDE）。

文章首先回顾了早期编程的情况，那时编写代码只是在白色背景上的黑色文本，编写代码需要大量的注意力，拼写错误并不会立即被突出显示，语法错误只会在编译时出现。然后文章介绍了 Asymetrix ToolBook，这是一款曾经因为多媒体而受欢迎，但现在已经被遗忘的工具。接下来，文章详细介绍了每一款 IDE，包括它们的特点、优点和影响。

6、把“你”放进CPU（英文）

这个网站是一个关于计算机处理器（CPU）的教学网站，它的目标是帮助读者理解当你在计算机上运行一个程序时究竟发生了什么。网站的创建者表示，他在计算机领域做了很多事情，但他一直对一个问题存在知识空白：当你在计算机上运行一个程序时，究竟发生了什么？他对这个问题进行了深入的研究，并分享了他的发现。

7、深度剖析 GPT 的原理、现状与前景

这篇文章是少数派的付费栏目「生产力超频：成为 ChatGPT 全域调教高手」中的一篇，主题是对 GPT（Generative Pre-trained Transformer）的深度剖析。

文章从 GPT 的原理、现状和前景三个方面进行了全面的探讨。文章首先解释了 GPT 的基本原理，通过一个形象的例子让读者对 GPT 有了初步的理解。然后，文章详细介绍了 GPT 的预训练过程，以及如何通过大量的数据训练出一个能够生成人类文本的模型。最后，文章对 GPT 的未来前景进行了展望，认为 GPT 将会对各行各业产生深远影响。

8、只有当你知道如何失败时，你才能成功（英文）

这篇文章强调了在做决策之前定义成功标准的重要性。尽管失败经常被美化，但没有人故意去追求失败。

每个人在创办公司、推出产品、聘请领导或购买房屋时都希望能够成功。文章指出，做出好决策的基础是在执行决策之前定义什么是好。成功的标准必须在项目开始之前定义，而不是在结束之后。

好奇星人

1、为什么拉完屎你需要回头看一眼 | 答案如下

这篇文章以幽默诙谐的方式，讲述了便便对我们健康的重要指标。

文章首先介绍了英国布里斯托皇家医院开发的便便形态量表，从最硬到最软的七种形态，其中第 3 到第 5 种被认为是正常形态。

然后，文章详细解释了便便的颜色、性状、细胞、虫卵、脂肪和细菌等各种指标对我们健康的影响。比如，便便中的血红蛋白浓度超过0.2μg/ml，检验报告上即会显示隐血：阳性。这可能是消化道出血的一个重要指标。

最后，文章强调了定期检查便便，这能够及时发现健康问题的重要性。

生态学中有一个“利基创造”的概念。这个想法是：并不是一个物种会来了，然后说，’哦，我可以在这里做得很好，那里有很多食物，’诸如此类。一个物种出现并做他的事情，通过在世界上行动，他创造了他生存所需的环境。

是你创造了需要的利基市场。这不像是世界在发展，’哦，真的需要约翰·希格斯的书，他们在哪里？但如果你这样做，世界就会对它们做出反应。

编码是一种创造行为，与写一首诗或画一幅画没有什么不同。然而，我们创造的美妙之处并不仅仅取决于我们算法的优雅或代码的效率，而在于其他人可以在我们的工作基础上建立的快乐和轻松。作为开发人员，我们的任务不仅仅是解决今天的问题，还要确保我们不会成为明天的问题。
